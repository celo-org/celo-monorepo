# Op-Deployer v3 — Upgrades to v2.0.0 & v3.0.0

Scripts for deploying and upgrading Optimism contracts to `v2.0.0` or `v3.0.0` on Celo networks using `op-deployer`.

## Prerequisites

- [op-deployer](https://github.com/celo-org/optimism) built from the appropriate branch
- `jq` installed for JSON processing

## Environment Variables

### Required Variables

- `VERSION` — Contracts version to deploy (`v2.0.0` or `v3.0.0`)
- `NETWORK` — Target network (`sepolia` or `mainnet`)
- `OP_ROOT` — Path to the Optimism repository
- `DEPLOYER_PK` — Private key of the deployer account

### Optional Variables

- `RPC_URL` — Defaults to `http://localhost:8545` (locally forked network). Set to target RPC for on-chain deployment.

## Scripts

### `run_upgrade.sh`

Orchestrates the complete upgrade process by running all scripts in sequence: `bootstrap.sh` → `bootstrap-validator.sh` → `upgrade.sh`.

**Features:**

- Automatically sets network-specific multisig addresses
- Falls back to default Anvil deployer key if `DEPLOYER_PK` is not set
- Validates `VERSION` and `NETWORK` before proceeding

**Network Multisig Addresses:**

- **sepolia**: `0x5e60d897Cd62588291656b54655e98ee73f0aabF`
- **mainnet**: `0x4092A77bAF58fef0309452cEaCb09221e556E112`

**Example:**

```bash
VERSION="v3.0.0" NETWORK="mainnet" OP_ROOT="/path/to/optimism" ./run_upgrade.sh
```

### `bootstrap.sh`

Deploys Optimism contract implementations and creates configuration files for validator and upgrade processes.

**Features:**

- Deploys all L1 contract implementations for the specified version
- Generates `config-validator.json` with deployed contract addresses
- Generates `config-upgrade.json` for upgrade configuration
- Supports network-specific proxy addresses and configurations
- Includes workaround for v3.0.0 deterministic address issues (`DelayedWETH`, `OptimismMintableERC20Factory`)

**Supported Versions:**

- **v2.0.0**: Prestate hash `0x03b357b30095022ecbb44ef00d1de19df39cf69ee92a60683a6be2c6f8fe6a3e`
- **v3.0.0**: Prestate hash `0x034b32d11f017711ce7122ac71d87b1c6cc73e10a0dbd957d8b27f6360acaf8f`

**Network-Specific Configurations:**

| Network | Superchain Config Upgradeable |
| ------- | ----------------------------- |
| sepolia | Yes                           |
| mainnet | No                            |

**Example:**

```bash
VERSION="v3.0.0" NETWORK="mainnet" OP_ROOT="/path/to/optimism" MULTISIG_ADDRESS="0x..." DEPLOYER_PK="0x..." ./bootstrap.sh
```

### `bootstrap-validator.sh`

Deploys the `StandardValidator` contract (required for superchain-ops) using the `config-validator.json` generated by `bootstrap.sh`.

**Supported Networks:** `sepolia`, `mainnet`

**Example:**

```bash
VERSION="v3.0.0" NETWORK="mainnet" OP_ROOT="/path/to/optimism" DEPLOYER_PK="0x..." ./bootstrap-validator.sh
```

### `upgrade.sh`

Performs the actual upgrade of L1 contracts to the specified version using `config-upgrade.json`.

**Features:**

- v2.0.0 upgrades require `--private-key` flag
- v3.0.0 upgrades do not require a private key (uses prank from config)

**Example:**

```bash
VERSION="v3.0.0" NETWORK="mainnet" OP_ROOT="/path/to/optimism" DEPLOYER_PK="0x..." ./upgrade.sh
```

## Configuration Files

### `config-validator.json`

Generated by `bootstrap.sh`. Contains deployed contract addresses for validator operations.

**Key Fields:**

- `release` — Optimism version
- `challenger` — Address of the challenger contract
- `l1PAOMultisig` — L1 Proxy Admin Owner multisig address
- Implementation addresses for all L1 contracts (AnchorStateRegistry, DelayedWETH, DisputeGameFactory, L1CrossDomainMessenger, L1ERC721Bridge, L1StandardBridge, MIPS, OptimismMintableERC20Factory, OptimismPortal, ProtocolVersions, SuperchainConfig, SystemConfig)

### `config-upgrade.json`

Generated by `bootstrap.sh`. Contains configuration for the upgrade step.

**Key Fields:**

- `prank` — Address to impersonate for upgrades (Proxy Admin Owner)
- `opcm` — Optimism Chain Manager address
- `chainConfigs` — Array of chain-specific configurations (`systemConfigProxy`, `proxyAdmin`, `absolutePrestate`)
- `upgradeSuperchainConfig` — Whether to upgrade superchain config

## Usage Workflow

1. **Export env vars:**

   ```bash
   export VERSION="v3.0.0"
   export NETWORK="mainnet"
   export OP_ROOT="/path/to/optimism"
   export DEPLOYER_PK="your_private_key"  # optional, defaults to Anvil key
   ```

2. **Run complete upgrade:**

   ```bash
   ./run_upgrade.sh
   ```

   Or run scripts individually:

   ```bash
   export MULTISIG_ADDRESS="0x4092A77bAF58fef0309452cEaCb09221e556E112"
   ./bootstrap.sh
   ./bootstrap-validator.sh
   ./upgrade.sh
   ```

## Notes

- The `run_upgrade.sh` script provides a safe default Anvil private key (`ac0974...`) for local testing
- Network-specific configurations are automatically applied based on the `NETWORK` environment variable
- Contract addresses are deterministic and network-specific
- The v3.0.0 bootstrap includes a workaround that reads previously generated `config-validator.json` to fix non-deterministic `DelayedWETH` and `OptimismMintableERC20Factory` addresses
